    在写这篇文章之前，其实我有无数次想要记录我这一脚踏入编程行业的道路，每天面对着电脑，面对着一行行红的绿的蓝的白的代码，早已冲淡我当年身上仅存的一些文艺气息，让我从半个文艺的斜杆女青年变成了左手键盘安天下，右手鼠标定乾坤的大码农。

    想说说自己为什么会取这样的一个标题，我是文科生出身，还在学校的时候，偏爱语文和地理，关于数学和英语，从小学一路差到大学，当真是佩服我这多年来对数英的不离不弃，当函数高数虐我千百遍的时候，我还挑灯夜战锁紧眉头苦思冥想费尽心思，两年前开始接触到前端的知识，18年七月份，整整过去了两年。

    这两年的时间里，学习前端知识体系，出来找工作，到在工作中独挑大梁，不断的学习前端新出来的框架，每天坚持写技术博客，拼命的努力提高自己各方面的能力，不敢有丝毫的怠慢，也害怕自己的才华配不上自己的野心和能力，无论何时何地都要做的最好，对得起公司每个月发的薪水。

    前面说了，每天上下班面对着代码久了，也在社会里面闯荡久了，思想也越来越功利和现实，面对这座城市的快节奏，面对这灯红酒绿之下的繁华与浮躁，内心动荡不安久久不能平静，再也做不到像从前那般，偏居一隅，雨天可以坐在窗户面前，临窗听雨，以手写心，静静的感受这时光的馈赠，午后的阳光看得见尘埃细细，上下沉浮，感受这片刻的静谧和安逸。

    反观现状，每天的思考时间会被挤散在早晚上下班高峰的人群里，每天的时间会被公司的领导占用，老板永远都不会嫌你干的活多，回到家的时间被淹没在油盐柴米酱醋茶的出租房琐事之中，业余时间也被各大头条、短视频、手机信息推送所断送，也许，生活便是如此，日复一日，年复一年，等到有一天才会幡然清醒，不知道已经这般浑浑噩噩过去了多久.

    人生在世，总要做些什么，总是要留下一些什么，才会心安，每个人都有自己的生活方式，前几天和一位做服务器高并发架构的大牛一起吃饭，在他的眼里，在他的人生之中，对技术之外事情都没有什么兴趣，从前到现在，一直如此，可以半路放弃高薪工作去选择自己的兴趣和爱好，可以做到工作三天三夜不合眼，觉得以看了一上午的高性能服务器架构相当于出国旅游一趟一样的精彩，他的专注和痴迷还有在乎的程度，让人佩服和榜样力量，这也是他的精神寄托和前进的方向、明灯、动力。

    所以，无论何时，不忘初心，方得始终。

    从小到大隐隐约约有写作的习惯，记录下那些好的不好的，开心的难过的事情，性格比较内向，从来不敢将自己的真实想法奉与台前供人翻阅查看评论，不知道从什么时候开始，也许是从决定踏进知乎的那一天，看着文章底下来来往往过路人的评论，有鼓励的，有尖酸刻薄的，有赞同的也有围观的群众，还有人发私信感谢我，感谢我的文章给了她帮助，给了她方向，似乎从那一刻起，坚持的动力又开始充满了意义。

    我不善于表达自己的感情，也没有多好的文笔，但正是来自大家的关怀和支持，正是这份信念，开始支撑着我每天去写文章，在很长一段时间里，在简书这个平台上，更新自己的技术博客，每天告诉自己，笔记还没有整理，文章还没有写完，还有人在等我，也正是如此，一天天过去，在这个过程中，我收获了很多很多，这大概是我这辈子都没有想到的事情

    1：认识很多志同道合的人

    在这个过程中，也遇到了很多同行，和他们交流的过程中，总是会发现一些积极向上的东西，每一个公众号主的背后，业余时间不是像大多数人一样，追剧，逛街，睡觉，玩游戏，而是在不断的读书，写作，思考和进步，他们有很强的号召力，也是很多粉丝的学习榜样，是一种在这个浮躁社会中可以静下心来学习的精神力量

    2：坚持的动力

    有些粉丝给我私信，发一些鼓励我的话语，有些人说，现在做技术的女孩子真的好少，看完你每天更新技术博客，真的很佩服你的毅力，这个时候的我，其实是美滋滋的，因为自己的付出有人能懂，能有肯定，心理总是会很温暖，很感动，自己的坚持的同事，也能给别人分享更多的知识和干货，当自己的文章能够解决了别人的问题和困惑的时候，也不算辜负每天在键盘上敲字的时间和精力。

    3：成就感

    运营公众号必不可少的就是原创文章，从前写一篇文章都不敢拿出来给别人看，因为我害怕被批评，害怕被别人指点，踏出这一步之后，我发现大多数的人还是友善的，他们会给我最好的鼓励，让我不忘初心，砥砺前行，看着过去自己写完的两三百篇文章，一种巨大的成就感油然而生，这比高中的时候成功做对了一道巨难的数学题还要开心。

    4：拓宽自己的人脉体系

    在做公众号之前，我的交际圈很小，天天上班面对着同事，下班面对着室友，朋友圈只有同学，在做公众号之后，我有了很多粉丝有了支持我走下去的人，有了很多同行，有前端工程师，Java工程师，Python，PHP等等工程师，也认识了一些公司的HR，it圈技术很厉害的大佬，运营圈写文章很受读者欢迎的人，还有很多合作伙伴，广告商，人脉体系在不断的完善起来。

    写了一个中午，没有休息，断断续续，其实我还是想要表达一件事情，专题一个文科妹子的前端悲欢编程之路建立起来之后，每周会更新一篇文章，写写这两年以及以后几年里我遇到的一些事情，写写我在it界的生存现状和感想，也顺便记录一下我对未来道路的一些思考和规划，专题可能有点长，在编程界的一天，这个专题就会不断的更新的，这也算是对自己的一种鼓励。

    想写一些关于vue的文章已经很久了，因为这个框架已经火了很久，在公司里用的框架都比较老旧，但怎么也得跟上前端发展变化的潮流，这不，开始使用vue开发项目了，一遍开发一边踩坑中，今天要记录的是五分钟搭建vue开发环境，运行GitHub上的开源项目。步骤很简单，只是在搭建环境的过程中会遇到各种各样奇怪的报错，十分让人头疼。

    官方文档：https://cn.vuejs.org/

    ###1：安装node
    端开发框架和环境都是需要 Node.js ，先安装node.js开发环境，vue的运行是要依赖于node的npm的管理工具来实现，下载https://nodejs.org/en/，安装完成之后，打开cmd开始输入命令。（我用的是win10系统，所以需要管理员权限，右键点击以管理员身份运行cmd），不然会出现很多报错。

    ###2：查看node的版本号
    下载好node之后，以管理员身份打开cmd管理工具，，输入 node -v ，回车，查看node版本号，出现版本号则说明安装成功。

    ###3：安装淘宝npm镜像
    由于npm是国外的，使用起来比较慢，我们这里使用淘宝的cnpm镜像来安装vue.
    淘宝的cnpm命令管理工具可以代替默认的npm管理工具。

    ###4：安装全局vue-cli脚手架
    淘宝镜像安装成功之后，我们就可以全局vue-cli脚手架，输入命令：cnpm install –global vue-cli 回车；验证是否安装成功，在命令输入vue，出来vue的信息，及说明安装成功；

    ###5：建一个新项目
    搭建完手脚架之后，我们要开始建一个新项目，这个时候我建议，尽量不要装在C盘，因为vue下载下来的文件比较大，如果要改盘的话，直接输入D：回车就可以直接改盘

    下面会出现是否需要js语法检测，这个我们暂时用不到，就可以直接输入no，后面的都可以直接输入no，都是我们暂时用不到的

    ###9：运行
    一切环境依赖安装准备就绪，我们来测试一下自己新建的vue项目的运行情况，输入命令：cnpm run dev直接回车。会弹出一个浏览器访问地址默认为localhost:8080。（我将地址修改为8081进行访问）
