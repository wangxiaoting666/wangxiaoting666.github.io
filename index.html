    一个前端妹子的悲欢编程之路

    在写这篇文章之前，其实我有无数次想要记录我这一脚踏入编程行业的道路，每天面对着电脑，面对着一行行红的绿的蓝的白的代码，早已冲淡我当年身上仅存的一些文艺气息，让我从半个文艺的斜杆女青年变成了左手键盘安天下，右手鼠标定乾坤的大码农。

    想说说自己为什么会取这样的一个标题，我是文科生出身，还在学校的时候，偏爱语文和地理，关于数学和英语，从小学一路差到大学，当真是佩服我这多年来对数英的不离不弃，当函数高数虐我千百遍的时候，我还挑灯夜战锁紧眉头苦思冥想费尽心思，两年前开始接触到前端的知识，18年七月份，整整过去了两年。

    这两年的时间里，学习前端知识体系，出来找工作，到在工作中独挑大梁，不断的学习前端新出来的框架，每天坚持写技术博客，拼命的努力提高自己各方面的能力，不敢有丝毫的怠慢，也害怕自己的才华配不上自己的野心和能力，无论何时何地都要做的最好，对得起公司每个月发的薪水。

    前面说了，每天上下班面对着代码久了，也在社会里面闯荡久了，思想也越来越功利和现实，面对这座城市的快节奏，面对这灯红酒绿之下的繁华与浮躁，内心动荡不安久久不能平静，再也做不到像从前那般，偏居一隅，雨天可以坐在窗户面前，临窗听雨，以手写心，静静的感受这时光的馈赠，午后的阳光看得见尘埃细细，上下沉浮，感受这片刻的静谧和安逸。

    反观现状，每天的思考时间会被挤散在早晚上下班高峰的人群里，每天的时间会被公司的领导占用，老板永远都不会嫌你干的活多，回到家的时间被淹没在油盐柴米酱醋茶的出租房琐事之中，业余时间也被各大头条、短视频、手机信息推送所断送，也许，生活便是如此，日复一日，年复一年，等到有一天才会幡然清醒，不知道已经这般浑浑噩噩过去了多久.

    人生在世，总要做些什么，总是要留下一些什么，才会心安，每个人都有自己的生活方式，前几天和一位做服务器高并发架构的大牛一起吃饭，在他的眼里，在他的人生之中，对技术之外事情都没有什么兴趣，从前到现在，一直如此，可以半路放弃高薪工作去选择自己的兴趣和爱好，可以做到工作三天三夜不合眼，觉得以看了一上午的高性能服务器架构相当于出国旅游一趟一样的精彩，他的专注和痴迷还有在乎的程度，让人佩服和榜样力量，这也是他的精神寄托和前进的方向、明灯、动力。

    所以，无论何时，不忘初心，方得始终。

    从小到大隐隐约约有写作的习惯，记录下那些好的不好的，开心的难过的事情，性格比较内向，从来不敢将自己的真实想法奉与台前供人翻阅查看评论，不知道从什么时候开始，也许是从决定踏进知乎的那一天，看着文章底下来来往往过路人的评论，有鼓励的，有尖酸刻薄的，有赞同的也有围观的群众，还有人发私信感谢我，感谢我的文章给了她帮助，给了她方向，似乎从那一刻起，坚持的动力又开始充满了意义。

    我不善于表达自己的感情，也没有多好的文笔，但正是来自大家的关怀和支持，正是这份信念，开始支撑着我每天去写文章，在很长一段时间里，在简书这个平台上，更新自己的技术博客，每天告诉自己，笔记还没有整理，文章还没有写完，还有人在等我，也正是如此，一天天过去，在这个过程中，我收获了很多很多，这大概是我这辈子都没有想到的事情

    1：认识很多志同道合的人

    在这个过程中，也遇到了很多同行，和他们交流的过程中，总是会发现一些积极向上的东西，每一个公众号主的背后，业余时间不是像大多数人一样，追剧，逛街，睡觉，玩游戏，而是在不断的读书，写作，思考和进步，他们有很强的号召力，也是很多粉丝的学习榜样，是一种在这个浮躁社会中可以静下心来学习的精神力量

    2：坚持的动力

    有些粉丝给我私信，发一些鼓励我的话语，有些人说，现在做技术的女孩子真的好少，看完你每天更新技术博客，真的很佩服你的毅力，这个时候的我，其实是美滋滋的，因为自己的付出有人能懂，能有肯定，心理总是会很温暖，很感动，自己的坚持的同事，也能给别人分享更多的知识和干货，当自己的文章能够解决了别人的问题和困惑的时候，也不算辜负每天在键盘上敲字的时间和精力。

    3：成就感

    运营公众号必不可少的就是原创文章，从前写一篇文章都不敢拿出来给别人看，因为我害怕被批评，害怕被别人指点，踏出这一步之后，我发现大多数的人还是友善的，他们会给我最好的鼓励，让我不忘初心，砥砺前行，看着过去自己写完的两三百篇文章，一种巨大的成就感油然而生，这比高中的时候成功做对了一道巨难的数学题还要开心。

    4：拓宽自己的人脉体系

    在做公众号之前，我的交际圈很小，天天上班面对着同事，下班面对着室友，朋友圈只有同学，在做公众号之后，我有了很多粉丝有了支持我走下去的人，有了很多同行，有前端工程师，Java工程师，Python，PHP等等工程师，也认识了一些公司的HR，it圈技术很厉害的大佬，运营圈写文章很受读者欢迎的人，还有很多合作伙伴，广告商，人脉体系在不断的完善起来。

    写了一个中午，没有休息，断断续续，其实我还是想要表达一件事情，专题一个文科妹子的前端悲欢编程之路建立起来之后，每周会更新一篇文章，写写这两年以及以后几年里我遇到的一些事情，写写我在it界的生存现状和感想，也顺便记录一下我对未来道路的一些思考和规划，专题可能有点长，在编程界的一天，这个专题就会不断的更新的，这也算是对自己的一种鼓励。

    2019年终前端总结（祈澈姑娘的技术学习篇）

    2019年已经快结束了，想起了年初的时候信誓旦旦立下的flag，现在回过头来在看一下，自己完成了多少了，2019年大前端学习指南（祈澈姑娘的学习规划）
    https://www.jianshu.com/p/5607e9a4719a
    ####语言语法层面
    ES6 ：学了vue和react之后，其实做项目的时候，会经常用到es6语法而不自知，只知道这样去用，却不知道具体的原理在哪里，后来周末花了两天的时间，集中看完了一套讲es6的语法的视频，并且写了系列的笔记，对ES6 的常用的语法有了一个大致的理解。

    TypeScript ：在学es6的同时，也顺便找了b站上点赞最高的TypeScript 视频，但是由于时间问题，只了解了数据类型，函数，类的定义继承等知识点，听说现在大厂都已经在用ts，所以学完语法已经迫在眉睫了，在使用Ant Design pro搭建自己的博客管理后台的时候，更加坚定了去学这个语法的决心，因为生成开箱即用的后台之后，发现全部用ts构建，很多都看不懂，用起来就变得非常的艰难了，就列入了1月份的学习名单。

    还有Rust，Dart，WebAssembly暂时用不到了，关于他觉得Flutter来做app是相当的nice的，尤其是作为Flutter的基本入门语法Dart，用一天的时间也就可以学会了，所以，也被列入清单啦。

    ####框架层面：

    Vue 2.0在很久之前就已经做过项目啦，3.0发布之后，只是稍微看了一下源码，没有去深入研究，搭建个项目玩一玩，目前处于观望状态，React 也很容易上手，只是jsx语法和平时的书写习惯有些不太一样，大概有些人用起来不习惯，但是Ant Design用起来相当的酸爽，文档组件一概齐全，复制黏贴分分钟就可以把项目模块都搞定。

    Angular 已经到8.0版本了，刚刚入行的时候才2.0，短短两三年的时间，更新换代速度太快了，已经远远把我甩在了身后，我也走上了从入门到放弃了，国内用的比较少，亦弃之。

    ####数据层：
    关于Redux ，GraphQL，要对自己说声抱歉了，年初立起来的数据层flag,没有用起来，公司的项目没有涉及到这些技术栈，（因为我不会，所以不用）尽管redux 可以用在任何框架中，

    ####JS 后端框架：
    在做博客的简易的登陆系统用到mongoDB存用户名密码等相关信息的时候，用的Node+express环境支撑做为后端框架，还是要重点熟悉一下Express和Koa，若想以后的前端之路能够发展全栈技能，这些都是绕不开的，需要作为重点培养对象。

    ####测试框架：
    使用分布比较匀称，Jest 和 Mocha ，说多了都是泪，这一块的flag可以打个零分了。

    ####跨端框架：
    Electron 和 React Native 排名比较靠前，受到广大开发者的喜爱和追捧。但也不得不说Flutter还是非常有必要去学，并且完成一个项目，最近使用的是uniapp，做的是公司的考勤app的项目，目前还没有遇到什么可以踩的坑，总体感觉良好。

    ####后端编程语言
    很早以前就学过了Python 的相关语法应用，也跟着校友很后面学了一些爬虫技巧，爬到网站的一些文章，图片等信息的最基本的功能，以前觉得一定要会一门后端语言，比如说php，现在的计划是，要延后两年了，先成为一个全栈，以后的事情再说吧，一个人得先学会爬再学会走路。
    ####构建工具和工具库：
    构建工具和工具库是经常要用到的，工具嘛拿来就用了，不需要花啥时间在上面耗，站在前人的肩膀上，行动起来非常的方便啦，Lodash木有去了解，Moment 现在一般用 ts日期工具: date-fns代替，函数式而且很轻巧，Moment 体积太大。

    ####浏览器 API：
    Service Workers、PWA、Web Components 这三个比较靠前，
    知道 Web Components 啥时候才能被大规模使用，反正 Polymer 看数据基本凉透了。
    PWA: 持续关注，不过这玩意其实真的很简单很简单，只要学会 Service Worker + Cache API 就算掌握了 80% 了，需要的时候再学也来得及。

    ###微信小程序:
    花了一周的时间跟着视频学完了小程序云开发，并且做了一款云开发的图书管理小程序，完全不需要后端的帮助，还是比较开心的。踩了一下小程序mpvue和wepy的坑，最后还是比较喜欢有赞的vant框架，适合做商城的项目，UI看起来也很舒服，（个人观点），听说Taro也还是不错的，有时间就去玩一下。

    ###其他热点技术
    还是那句话，前端的技术热点太多了，技术更新换代的速度很快，各种框架层出不穷，如果有时间有精力的话，可以选择一些自己喜欢的去做。

    ###总结和展望
    好啦，说了那么多，开始总结了，2019年给自己立下的flag，其实完成程度一半还没有到，（虽然已经很努力的在学习啦）接下来在新的一年里，重点需要发展：

    #####重点学习的部分：
    Dart
    Vue 3.0
    TypeScript
    Redux
    Koa
    React Native
    Flutter

    #####技术栈做一些项目：
    React Native移动端：电影平台项目
    Flutter：高仿马蜂窝项目
    小程序：做一个商城项目，框架暂定
    uniapp：做一个考勤app的项目
    博客：Ant Design做管理后台展示界面，Node.js + MongoDB做后端支持

    面试经历|为即将找工作的你保驾护航

    聊一聊我的面试经历，那年数九寒冬，坐标北京，初冬的第一场雪，零下十几度。

    如果你也是第一次找工作，不要忧伤不要焦虑，因为有一天，你也会像我现在这样，上班时间，开着微信和扣扣，该聊天时就聊天，像极一个老油条。

    开始找工作的时候是双十一，当大家都沉浸在剁手的气氛里，我冒着北京的寒风，开启我的求职之路，我记得在智联和拉钩上投了一百来份简历，却在第二天中午收到面试通知，滴滴打车，下午两点面试，我连饭都来不及吃飞奔出门，却无奈在偌大的北京城，穿城而过密密麻麻的地铁里迷了路，下地铁的时候，已经错过了面试的时间。

    我站在地铁口的寒风里，不知所措。

    接下来的几天里，基本上都没有面试，我很焦急，打电话给同学给老师给学长学姐，终于总结出原因，第一，简历写的太过于简单，第二，投的简历太少。

    简历里面不能少了三样东西，知识技能，工作经验和项目经验，其余的都无关紧要，如何能让hr在众多简历之中一眼能相中你的简历并通知面试，需要的还是一定的功夫，那就是完善自己的简历。

    简历完善之后，每天向智联上投五百份简历，终于，平均每天有了两到三个面试电话，当时正值年末，很多企业都已经不再招人，所以岗位少之又少，前来面试的人却很多，一度呈现僧多肉少的局面，竞争激烈。找工作最好的时间是三月到四月，（年初职位变动大，招聘岗位多）

    面试大多数有两轮，笔试和面试，有时候还会有机试和复试，笔试的题目大多数都是出自百度，面试问的问题也都是一些基础常见的，更多偏向于原生，或许是自己的技术不够硬，面试了七八家之后，都在一句等通知里面结束。

    我疑惑，于是又根据前几家面试时候遇到的事情，总结了一下面试中出现的问题，并采取了一系列的措施。

    面试之中的坑：

    1.不负责任的公司

    这些公司之中，有几家公司特别坑，有一家公司明明是招UI设计师，却也通知我去面试，等到了现场之后，他才发现不对，于是一句话打发走。明明很气愤，却还要强忍着愤怒，公司只需要一个电话的通知，而我就要连早餐都来不及吃，倒公交换地铁冒着刺骨的寒冷一路导航过去，这些都不会被高高在上的公司人事或主管所知道，他们的一句话，决定着多少人的命运。

    2.做页面

    这种公司直接pass掉吧，太过分的就是这样的公司，自己没钱招人，就打着招聘的幌子，把出的UI图挂起来，谁做完就考虑招进来，前去面试，就有个UI模样的设计师坐在我的对面，打开电脑搜索面试题，对照着这些面试题一题一题的问，问完了还来一句，哦，我觉得你很不错很专业，但是我们公司有个规定，我发几个页面给你，你做完发我邮箱，我看看可不可行，如果满意的话，我们就考虑通知你过来上班了，这些个页面少说也得做一天，太浪费时间，出门的时候碰到了别的面试者，他们也说遇到了同样的事情，那万一不录取？是不是还得支付我一天的工钱？

    3.培训机构

    往往会有很多培训机构打电话邀请前去面试，在智联上挂号，有模有样有薪资有待遇，但到公司一看，实际上要么是个大型的培训机构，要么是小型培训机构，当时是一个特别机灵的女生来和我聊，她开门见山说，我这个人不喜欢拐弯末脚，我更喜欢坦诚布公，希望你能如实回答，那凌厉的眼神似乎我真的有了什么不可告人的秘密，我淡定的说了句好。等一切说完，她直接说，你知道的，我们的条件就是，你跟在我们公司后面学习，一起做项目，半年过后，如果你表现好的话，我会把你推荐到一家好的企业，但是，天下没有白吃的午餐，你的给我报酬，等你上班之后，每个月得给我们四千块钱吧，我没等她说完就走了，冒着风雪来一趟还真不容易，心凉了半截。

    那如何区别这样的公司？

    智联上很多这种培训机构，最大的区别就是，没有具体的技术要求，一般按理说，没有任职要求的公司，就不用白跑一趟了。

    4.初创公司

    注意我说的是初创公司而不是创业公司，像这种公司就拉入黑名单吧，公司就一个老板一个人事，我就遇到这样坑爹的公司，喊一堆人过去面试，平均每个人三到五分钟，问的问题是身高年龄体重籍贯家庭住址居住地，最后再来一句要是让你做一个系包括后台数据库，你可以吗？这不是在逗我？连基本的开发团队都不知道吗？要求一个前端什么都会那你为什么不去招一个全栈？

    其实当时没觉得有什么，但是，后来又发了面试邀请让我去面试，我打电话问人事是不是复试，人事简单敷衍说我来看看稍后给你回电话回复你这是什么情况，然后就没有然后了，再打电话就没人接了，但求职心切的我还是去了公司，跟上次一样，上来就是重复了一模一样的问题，我晕，我说既然不是复试你们叫我来是什么意思，人事尴尬的把责任全部都推到我的身上，因为你在智联上留了两个号码吧，上次跟这次通知的是两个号码，睁着眼睛说瞎话，我恼怒了，到底录不录用你给一句痛快话，她们不以为然并且丢下一句话，我们会给你通知的。

    对于这样不尊重人的公司，我只想说，拉黑。

    5.烂摊子

    你再生气也得忍着，因为他们都是大爷，还有一种公司，急需招人，只要你有点技术可以干活，就很容易收到面试邀请，但是，公司的情况大多数都是这样的，前任工程师离职，离职原因不详，留下了一个未完成的项目，现在要招人继续编写开发，其实就是个坑，进去了也不一定能完成，圈内有个笑话，程序员为什么要离职，因为他发现，自己挖的坑已经补不上了，再来一个人，那八九不离十就是入坑了，珍爱生命，慎重选择。

    面试了十八家公司，带着麻木和绝望离开北京，带着刻骨的寒冷和绝望，站在北京西站古老的城楼之下，北京，再见。

    十二月初，辗转来到上海，这里气候温暖，是种完全没有入秋的温暖，三天时间，拿到了两个offer，也许是上天眷顾，也许是我和这座城市有缘，至此，结束了我漫长的求职生涯。

    终于等到你，幸好我没放弃。

    面试前的准备

    1：简历

    简历一定要精细化，精通那些又熟悉哪些，程度的区分是非常讲究的，前两天把在北京求职的简历挂在知乎上，引来不少人的批评和改正，写简历是一门学问，不是胡乱写写就行的，资深hr一眼便能识别出简历的好坏，是千篇一律还是别具一格，从技术，知识点，精简程度，排版，都能很好的反应出一个工程师的修养和水平。

    2：投放时间和数量

    下面是一组智联招聘上显示的数据，所有行业之中最激烈的，前端排名第二，这就意味着一家公司招聘，邮箱里分分钟就会被求职者塞满简历，所以投简历一般要选择在hr早晨上班或者喝下午茶的时间，那个时间段她们正好有空打开邮箱，而幸运的你，投递及时，简历就会被查看，被邀请面试的机会就很大，很多人有这样一个疑问，为什么我的简历大多数都石沉大海，没人回应了？

    那是因为，投的数量太少，当你像我一样每天海投五百份的时候，就不会存在这种疑问咯。

    3.背面试题

    前面说过，严格的公司是有四轮面试的，笔试，机试，面试，复试，如何能在层层筛选之中脱颖而出，如果技术不是很好，短时间之内是没有办法提高的，那么只能像大学的期末考试一样，应付挂科的办法，那就是背老师划得重点咯，找工作也是一样的道理，百度总结出网上所有能找到的前端面试题，开启死记硬背的模式，每天晚上背到凌晨四点钟，不管是笔试还是一般的技术官前来面试，那都是对答如流

    机会，总是留给做好准备的人。

    4.能够独挡一面

    公司需要的是能够干活的人，如果你可以一个人干完三个人的活，那老板必定是坐在办公室里面偷着乐，我们公司产品总监有一句口头禅，技术上的困难都不是困难，没有什么问题是技术不能解决的。

    或许此时此刻的你没有这份自信，觉得自己一个人完成不了一个项目，但事实是，进入公司之后，会遇到种种困难，从接手一个项目开始，就相当于是你决定了要攀爬一座山峰，刚刚开始的路还算平坦，越往上爬，会遇到荆棘，丛林，陡崖峭壁，猛虎野兽，但是不能回头，更不能半途而废，只能逼着自己往前走。因为没有退路。

    当你历尽艰难登上山顶的时候，再次回头，是什么感觉？是征服激动和喜悦。

    那么恭喜你。

    5.自己的过人之处

    在别的文字里面我也提过，作为一名程序员，大多数人的身上都是有不断探索的精神，这也得益于平时bug的磨练，每个人都有一套属于自己进步的方法，这里，我就不再过多的描述，欢迎评论留言，留下你在日常工作中与众不同的技巧和方法。
